## έγκ TL;DR

A pretrained LLM is enormous, expensive, and frozen. **Fine-tuning** is the art of adapting it έΑΦ efficiently, on consumer hardware, without forgetting what it already knew. This repo covers it end-to-end: the **theory** (a structured book of 8 chapters) and the **practice** (working code that fine-tunes real VLMs on real datasets and beats H100 baselines on a $300 GPU).


## ΏθΥΨ Part 1 έΑΦ The Book (`library/`)

An **Obsidian vault** with a structured book on fine-tuning of LLMs, based on articles from "The Neural Maze ┬╖ Finetuning Sessions". Each chapter teaches with concrete numerical examples, analogies, and step-by-step formula breakdowns έΑΦ not a translation of the source article.

| # | Chapter | What it covers |
|---|---|---|
| 1 | **Foundations** | Transformer architecture, attention, pretraining. The base needed for everything else. |
| 2 | **SFT** (Supervised Fine-Tuning) | Teaching the model to follow instructions. Chat templates, loss masking, chain-of-thought. |
| 3 | **LoRA** (Low-Rank Adaptation) | PEFT: train tiny low-rank adapters instead of moving billions of weights. The intuition + the math. |
| 4 | **QLoRA** | LoRA on 4-bit quantized models. NF4, double quantization, paged optimizer. Frontier models on consumer GPUs. |
| 5 | **RLHF** | Aligning with human preferences έΑΦ what SFT alone can't teach. Reward models, PPO, DPO, KL divergence. |
| 6 | **GRPO and variants** | The RL algorithm behind DeepSeek's reasoning models. PPO without a critic. Includes DAPO, GSPO, Dr GRPO. |
| 7 | **Multimodal** | Beyond text: Vision-Language Models (VLM) and Text-to-Speech (TTS) with neural audio codecs. |
| 8 | **Inference at scale** | Once trained, how do you serve it? Prefill/decode, KV-cache, paged attention, vLLM, continuous batching. |

The book is generated by a chain of Claude Code agents (`redactor`, `visualizador`, `scanner`, `enlazador`, `generador-conceptos`, `unificador`) that turn a technical article into a linked, illustrated chapter and finally export the whole thing to Word.

**To open the book:**
```bash
cd library
claude   # the book has its own .claude/ with specialized agents
```

Where the theory becomes runnable code. Real fine-tuning experiments on real datasets, designed to **run on a single consumer GPU** (RTX 3060 12 GB) using `Unsloth` + `PEFT` + `TRL`.

| Project | What it is | Status |
|---|---|---|
| [`code/wildfire-prevention/`](./code/wildfire-prevention/) | Fine-tune compact VLMs (LFM2.5-VL, Qwen2.5-VL, Gemma-3) to predict wildfire risk from Sentinel-2 imagery έΗΤ JSON. Comparative study across 4 model families. | έεΖ Working |
| `code/docs/` | Reusable docs that apply to any project: WSL2 setup, SFT training guide (epochs, hyperparams, what-good-looks-like). | έεΖ Working |

### Highlights from the wildfire project

- **Beat the original Liquid AI cookbook** (which uses full fine-tuning on H100) with **LoRA on a 12 GB consumer GPU**.
- Side-by-side benchmarks of 4 VLM families with consistent hyperparameters.
- Full pipeline: dataset cloning έΗΤ JSONL preparation έΗΤ LoRA training έΗΤ evaluation έΗΤ comparative report with plots.

## ΏθΤκ Design principle

> **Theory and practice in the same repo, but in clean rooms.** The book teaches *why* a technique works. The code lets you *try* the technique. You can read a chapter, jump to the corresponding code, train something, and come back to the chapter with sharper questions.

That's what this repo is for.
